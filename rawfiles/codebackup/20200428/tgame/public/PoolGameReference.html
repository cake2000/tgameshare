Functions for Game Bot

Function Summary

isPathBlocked(p1, p2) Check if a ball can roll from p1 to p2 without hitting any blocking ball.
dist2(p1, p2) Measure the distance between the points p1 and p2.
calculateProbability(cmd) Calculate the probability for successfully hitting the target ball into the target pocket, given the shot command cmd.
calculateCutAngle(ball_id, pocket_id) Calculate the cut angle for shooting the ball (whose ID is ball_id) into the pocket (whose ID is pocket_id).
calculateSidePocketSkew(ball_id, pocket_id) Calculate the skew of the ball (whose ID is ball_id) against the side pocket (whose ID is pocket_id).
calculateCTP(cmd, ball_id, target_point) Calculate the CTP (Closest Trajectory Point) for the ball (whose ID is ball_id) against the target_point: given this command, project the trajectory of this ball, and find the point where this ball is closest to the given target point.
getSecondsLeft() Query the game engine for how many seconds are left for your gamebot.
calculateEndState(cmd) Query the game engine for the end state of this command, which tells you the end position for every ball.

Detailed Description

Balls

Variable back to top

Description

An array of ball information objects, with each object containing the following properties:
ID
The ID of the ball, from 0 to 11 for the Beginner Level game, or from 0 to 21 for the Advanced Level game. Note that the cue ball's ID is 0, and the black ball's ID is 1. So for example, you can get the cue ball's coordinates using Balls[0].x and Balls[0].x
inPocket
A boolean indicating whether this ball is in some pocket already (false means it is still on the table).
colorType
The color of the ball: 0 for "red", 1 for "yellow", 2 for "white" (the cue ball), and 3 for "black".
x
The x coordinate of this ball (invalid if inPocket is true).
y
The y coordinate of this ball (invalid if inPocket is true).

MyColorType

Variable back to top

Description

The color type of balls that my game bot can legally shoot, either because my game bot has chosen that color, or because its opponent has chosen the opposite color.

Possible values: null if no color has been chosen yet (still in Open Table mode), 0 for red balls, 1 for yellow balls.

 

 

BallDiameter

Variable back to top

Description

The diameter of each ball (all balls are of the same size). It is currently set to 41.666667, but it is subject to future changes, so you should use the BallDiameter variable instead of using this number directly.

A ball's diameter is 2 times its radius, so when 2 balls are touching each other, the distance between their centers is exactly the ball diameter.

Illustration

 

 

TOP_Y, BOTTOM_Y, LEFT_X, RIGHT_X

Variable back to top

Description

The y and x coordinates of the 4 rebound lines at the 4 cushions.

As illustrated, these 4 blue lines are the rebound lines for the 4 cushions. For any ball to rebound off a cushion, it will be touching that cushion for a brief moment, and the ball center's distance to that cushion is exactly the ball's radius. Therefore, we can use these lines to calculate the mirror point of any target point that we want to hit after a rebound.

For example, for a target point at (100, 200), if we want to know its "mirror point" against the top cushion, then we can use the TOP_Y this way: mirrorPoint.y = 2*TOP_Y - 200.

Illustration

dist2(p1, p2)

Game Bot Function back to top

Example Usage

const distance = dist2({x: 10, y: 20}, {x: 30, y: 40});

Parameters

p1
The coordinates of the first point, with properties "x" and "y".
p2
The coordinates of the second point, with properties "x" and "y".

Return value

The distance between p1 and p2. For example, if p1 is at (3, 5), p2 is at (7, 8), then their distance is the square root of (7-3)x(7-3)+(8-5)x(8-5), which equals 5.

 

 

calculateProbability(cmd)

Game Bot Function back to top

Example Usage

const cmd = {aimx: 20, aimy: 30, strength: 40, targetBallID: 5, targetPocketID: 2};
const probability = await calculateProbability(cmd);

Parameters

cmd
The shot command, with the following properties:
targetBallID (required): the ID of the target ball.
targetPocketID (required): the ID of the target pocket (0 to 5).
aimx: the x coordinate of the aiming point for the cue stick, with a default value of 0 if not provided by the game bot.
aimx: the y coordinate of the aiming point for the cue stick, with a default value of 0 if not provided by the game bot.
strength: the strength for the shot, between 2 and 100, with a default value of 30 if not provided by the game bot.
spin: the vertical spin for the shot, between -1 (full back spin) and 1 (full forward spin), with a default value of 0 (no vertical spin) if not provided by the game bot.
hspin: the horizontal spin for the shot, between -30 (30 degrees to the right) and 30 (30 degrees to the left), with a default value of 0 (no horizontal spin) if not provided by the game bot.

Return value

The probability of successfully hitting the target ball into the target pocket, considering shot misses due to randomness in shot direction. The range of random error increases if more strength or spin (horizontal and vertical) are applied.

The value range is from 0 to 100, which represents probability of 0 to 100%.

 

 

calculateCutAngle(ball_id, pocket_id)

Game Bot Function back to top

Example Usage

const cutAngle = calculateCutAngle(5, 1);

Parameters

ball_id
The ID of the target ball
pocket_id
The ID of the target target

Return value

The cut angle for hitting the target ball into the target pocket.

For example, as illustrated below, to calculate the cut angle for shooting ball 8 into pocket 3, you can call "calculateCutAngle(8, 3)".

Internally, 2 lines are drawn: one from ball 8 to pocket 3, and another from the cue ball to the aiming point. The absolute value of the angle between these 2 lines is returned, which is about 41 degrees in this case.

Illustration

 

 

calculateSidePocketSkew(ball_id, pocket_id)

Game Bot Function back to top

Example Usage

const skewAngle = calculateSidePocketSkew(5, 1);

Parameters

ball_id
The ID of the target ball
pocket_id
The ID of the target target

Return value

The absolute angle between 2 lines: one from the pocket to the ball, another from the pocket vertically up or down. If the pocket_id is not 1 or 4, this function will return -1.

For example, as illustrated below, to calculate the skew angle for ball 2 against the side pocket 1, you can call "calculateSidePocketSkew(2, 1)".

Internally, 2 lines are drawn: one from pocket 1 to ball 2, and another from the side pocket 1 vertically down. The absolute value of the angle between these 2 lines is returned, which is about 24 degrees in this case.

Illustration

 

 

calculateCTP(cmd, ball_id, target_point)

Game Bot Function back to top

Example Usage

const cmd = {aimx: 20, aimy: 30, strength: 40, targetBallID: 5, targetPocketID: 2};
const CTP = await calculateCTP(cmd, 4, {x: 20, y: 30});

Parameters

cmd
The shot command to be used for trajectory projection.
ball_id
The ID of the ball whose trajectory will be examined
target_point
The target point against which we want to find the closest trajectory point.

Return value

The coordinates (an object with "x" and "y" properties) of a point on the trajectory of this ball, such that through out the trajectory, the ball is closest to the target point at this location.

For example, as illustrated below, given this trajectory (projected using the given command), the cue ball (ball_id == 0) is closest to the blue target point at the CTP shown as a green circle.

Usually the CTP is used to measure how much the ball is missing the target point, and we can adjust the shot command based on the relative position of the CTP from the target point. For example, if the CTP is to the right of the target point, we can adjust the aiming point of the shot leftwards, which should bring the trajectory closer to the target point.

calculateEndState(cmd)

Game Bot Function back to top

Example Usage

const cmd = {aimx: 20, aimy: 30, strength: 40, targetBallID: 5, targetPocketID: 2};
const endState = await calculateEndState(cmd);

Parameters

cmd
The shot command to be used for trajectory projection.

Return value

The end state of all balls, which is an array of objects. Each object contains up to these 4 properties:

ball_id
The ID of the ball represented by this object.
inPocketID
If this ball has already fell into a pocket, then inPocketID is the ID of that pocket, which can be 0 to 5. If this ball is still on the table, then inPocketID will be null.
x
If this ball has already fell into a pocket, then x is undefined. If this ball is still on the table, then x will be the x coordinate of this ball.
y
If this ball has already fell into a pocket, then y is undefined. If this ball is still on the table, then y will be the y coordinate of this ball.

Note that the end state is only describing the base case trajectory of all balls after a shot, without considering any random noise associated with the shot. It is equivalent to the baseline trajectory you see when you adjust your shot manually.

For example, for this shot illustrated below, the end state returned by the game engine will be an array like this:

            {ball_id: 0, inPocketID: 5},
            {ball_id: 1, inPocketID: null, x: 780, y: -137},
            {ball_id: 2, inPocketID: 0},
            {ball_id: 3, inPocketID: null, x: 607, y: -353}, 
            {ball_id: 4, inPocketID: 0},
            {ball_id: 5, inPocketID: 0},
            {ball_id: 6, inPocketID: 0},
            {ball_id: 7, inPocketID: 0},
            {ball_id: 8, inPocketID: 0},
            {ball_id: 9, inPocketID: 0},

            {ball_id: 11, inPocketID: 0},
          ]

ChooseRedColor() or ChooseYellowColor()

Test Script Function back to top

Example Usage

ChooseRedColor();

Parameters

none

Purpose

Set the color of legal balls your gamebot can shoot to red or yellow. If you don't call this function in your test script, your gamebot's MyColorType will be null, so your gamebot can choose to target any red or yellow ball. After you call this function, MyColorType will be set to 0 for red balls or 1 for yellow balls.

ExpectBallPocketed(ball_id, pocket_id)

Test Script Function back to top

Example Usage

ExpectBallPocketed(4, 5);

Parameters

ball_id
The ID of the ball that you expect to be pocketed after all balls have stopped. The value can be between 0 and 21.
pocket_id
The ID of the pocket that you expect the ball to fall into, which can be 0 to 5.

Purpose

Tell the game engine that you expect the specified ball to be pocketed in the specified pocket. The game engine will verify if that ball has actually fallen into that pocket, and report this test as passing if so.

 

 

ExpectShotCommand(command)

Test Script Function back to top

Example Usage

const cmd = {aimx: -828, aimy: 80, strength: 40, spin: 0, hspin: 0, targetBallID: 4, targetPocketID: 5}; ExpectShotCommand(cmd);

Parameters

command
The command that you expect your gamebot to return from its getCallShot function. It could contain any of the 7 basic properties of a shot command: aimx, aimy, strength, spin, hspin, targetBallID, targetPocketID.

Purpose

Tell the game engine that you expect the command returned by your gamebot's getCallShot function to match with the command you have specified here, and report this test as passing if so.

For example, if the command you have specified has a property "aimx" as 200, but the shot command returned by your gamebot has an aimx of 300, then this test will fail.

Note that if you do not specify a property, such as "strength", in your expected command, then the game engine will not try to match that property with the actual shot command returned. For example, if you only want to make sure your gamebot returns the correct aiming point, then you only need to specify "aimx" and "aimy" of the expected command in the test script.
