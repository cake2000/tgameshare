A JavaScript function is a block of code designed to perform a particular task or calculate a value, like these two examples below. It can take some optional input parameters, and it can optionally return some object or value as output.

// syntax 1
function add(x, y) { 
    return x + y;
}
// syntax 2
let subtract = function(x, y) {
    return x - y;
};

A JavaScript function executes when you call a function by appending a parameter list (can be an empty list) to it like this: 

// call the add function with 2 parameters and assign 
// the output to the variable result
let result = add(2, 3);

// call subtract function with 2 parameters
// and print the result using the function console.log
console.log(subtract(result, 2));

async and await are used together to wait on an asynchronous task to finish before continuing. For example, setTimeout is a commonly used asynchronous function: it says please run this function after this many milliseconds. Without using async and await, a JavaScript program runs "through" asynchronous calls like this: 

function run1() {        
  // print "out 1"
  console.log("out 1");
  // wait 1000 milliseconds and then print "out 2"
  setTimeout(function () {
    console.log("out 2 ");
  }, 1000);
  // print "out 3"
  console.log("out 3");
}

run1();

// output: 
//out 1
//out 3 
// after 1 second
//out 2

As you can see, since setTimeout is an asynchronous function, the execution continues passing the setTimeout line to print "out 3" first, then after 1 second the "out 2" line is printed.

If we want the execution to block (wait) at the setTimeout clause, we need to a) mark the function as async, b) wrap the setTimeout call in a Promise, and c) use await to call the new timeout function, like this: 

function timeout(ms) {
  // "wrap" the asynchronous function 'setTimeout' in a promise
  return new Promise(resolve => setTimeout(function () {
    console.log("out 2 ");
    resolve();
  }, ms));
}            
async function run2() {    
  // print "out 1"        
  console.log("out 1");
  // call async function to print "out 2"
  await timeout(1000);
  // print "out 3"
  console.log("out 3");
}

run2();

// output: 
//out 1
//out 2
//out 3

Cool Trick: you can actually test-run both programs on any browser's developer console. For example, in Chrome or FireFox, press F12, and select the "Console" tab on top, then you can copy and paste the example code above to test it.

In JavaScript, the keyword this can be confusing sometimes. For majority of use cases, it can be simplified to one of the following two scenarios:

1. When this is used inside a function called through an object, it refers to that object. For example:

const obj = {
  name: "object1",
  printName: function() {
    console.log(this.name);
  }
};
// invoking function through obj -> will print 'object1'
obj.printName();

2. Otherwise, you can assume this is referring to the global object window. For example:

const printName = function() {
  console.log(this.name);
};
// invoking function directly instead of through an object
printName(); // will print window.name if that's defined

In real life, to achieve a random effect, you can throw a coin and choose different actions based on which side of the coin is facing up. We have something much more powerful in the computer world: let r = Math.random(). This is a function of the Math libray, and every time you call it, it returns a new random number r between 0 and 1.

So to simulate a coin flip, you just need to keep generating new random numbers, and call it a "head" whenever it is greater than 0.5, since there is a 50% chance r is greater than 0.5.

If you keep drawing random numbers using Math.random, it gives you a series of simulated coin tosses like this:
Math.random outputs: 0.34 0.58 0.79 0.13 0.80
simulated coin tosses:

Of course, the threshold doesn't have to be exactly 0.5. For example, if you want to simulate a random event that happens with 20% probability, you can keep drawing random numbers using Math.random(), and trigger the event each time you get a number that's greater or equal to 0.8 (or less than 0.2).

To understand the for-loop, let's consider the simple task of listing all the even numbers from 0 to 6. The approach is fairly straight-forward: you start with 0 and add 2 each time, until you reach 6.

Now let's code this repetition in a for-loop. As shown below, a for-loop is composed of three key components: initialization, test condition and change step, followed by the block of code to be repeated. Each repetition of the code is called an iteration.

  • initialization defines a new index varaible using the keyword let (executed before the first iteration starts);
  • test condition is checked before each iteration, and no further iterations will be executed after the condition becomes false;
  • change step runs at the end of each iteration;

As illustrated, we first initialize the index variable, i, to 0. (Note that you can use any variable for the index: "k" or "j" for example). Next, we run the current iteration, which prints "i is 0".

Then the index variable i is increased by 2 with "i = i + 2". Note that when we write "i = i + 2", we first calculate the value of "i+2" to get the value of 2, then assign that value to the variable on the left of the equation, which is i itself. Now we check the test condition i <= 6 again. Since the condition is true, we stay in the loop and "i is 2" is printed.

On the third and fourth iterations, "i is 4" and then "i is 6" are printed. When i is finally increased to 8, the stopping condition "i <= 6" no longer holds, so we exit the for-loop at that point.

When using a for loop or a while loop, if you want to stop the loop and break out right away, you can use the keyword break like this example below. Note that when i becomes 6, the break keyword gets called, and the program exits the for-loop right away. 

for (let i=0; i < 10; i++) {
  if (i > 5) {
    break;
  }
  console.log("iterator i is " + i);
}
// will print 
// iterator i is 0 
// iterator i is 1 
// iterator i is 2 
// iterator i is 3
// iterator i is 4 
// iterator i is 5

When using a for loop or a while loop, if you want to skip the current iteration and jump to the next iteration right away, you can use the keyword continue like this example below. Note that when i becomes 3, the continue keyword gets called, and the program moves on to the next iteration right away. 

for (let i=0; i < 5; i++) {
  if (i == 3) {
    continue;
  }
  console.log("iterator i is " + i);
}
// will print 
// iterator i is 0 
// iterator i is 1 
// iterator i is 2 
// iterator i is 4

Grid search is a basic algorithm used to search through a list of items and select one best item. Here best is specified by you, which is usually based on some numeric value associated with each item.

For example, imagine you are choosing a hero to use in a fighting game. There are 3 heroes to choose from, and each hero has a damage level and a speed level: 

var heroes = [];
heroes.push({name: "FireMaster", damage: 6, speed: 2});
heroes.push({name: "DragonKing", damage: 4, speed: 5});
heroes.push({name: "SnowRunner", damage: 3, speed: 4});

Now, assume you want to search for one hero that has the highest combined damage and speed levels, then you can write a simple grid search like this: 

// record keeper for best combined level              
let bestLevel = 0;
// a pointer to the best hero so far as you iterate
let bestHero = {};                
// for-loop to iterate through all heros in array
for(let i=0; i < heroes.length;i++) {
  // calculate level to be used for comparison
  let combinedLevel=heroes[i].damage+heroes[i].speed;
  // if level is better than best so far
  if (combinedLevel > bestLevel) {
    // update best level record
    bestLevel = combinedLevel;
    // update pointer to point to current hero
    bestHero = heroes[i];
  }
}

Note that since we are looking for a hero with the best combined level, we needed a variable bestLevel to keep track of the best level we have seen so far, and use it to compare with each new hero down the list. Also, whenever we have found a new best, we need to keep track of that hero using a separate variable bestHero. In our little example, the bestHero will be pointing to the DragonKing, as his combined level is at 9, the highest.

In JavaScript, let declares a new variable, which is a container for you to store a value that you can read back later. Unlike const, you can assign a new value to it as well. For example: 

let x = 1;
let y = 2;
x = y + 3; // x is 5 now

// variable's value can be a string
let w = "abc"; 
console.log("w is " + w);

// variable's value can be a function
let subtract = function(a, b) {
    return a - b;
};
let result = subtract(x, y);

In JavaScript, const declares a new constant container that will not be reassigned later. For example: 

const x = 1;
const y = {
    a: 2
};
// you can change the property of y
y.a = y.a + 1;

// but you cannot reassign y
y = {
    b: 3
};

A JavaScript object is used to represent a real-world object in the computer world, such as a car or a movie.

When we describe a real-world object, we can talk about both its attributes and its behavior.
Each attribute has a name and a value: a car's color is red, speed is 50; a movie's title is Frozen, length is 120 minutes. These attributes can be represented in JavaScript like this: 

let car = {
    color: "red",
    speed: 50
};
let movie = {
    title: "Frozen"
};                
// you can attach new attributes to objects 
movie.length = 120;
Behavior is more about what the object can do. When a car runs, its position is changed as a result. Some behaviors are more abstract. For example, a movie can rename itself by changing its title. These behaviors can be represented in JavaScript as method functions attached to the object like this: 
let car = {
    color: "red",
    speed: 50,
    // define behavior as part of object creation
    run: function() {
        // change position of car
    }
};
// trigger a method
car.run();

let movie = {
    title: "Frozen",
    length: 120
};            
// attach new behavior to object 
movie.rename = function(newTitle) {
    this.title = newTitle;
}

There are a few key facts to remember about objects:

  • An object can be empty, as represented by {}.
  • It's perfectly fine for an object to have no method function defined, as we often just need to store some attributes in an object.
  • The value of an attribute can be another object in itself. For example, a car's wheel can be an object itself: 
                            car.wheel = {
                            diameter: 17, color: "silver"
                        };

A JavaScript object (called JSON) is a collection of properties, where each property is a map from a key to a value.

JSON is very useful when you want to define a map from string type keys to their values, or when you want to represent an object with some properties.

Example:

var obj = {
    // the value can be any simple "primitive" value
    key1: "string value", 
    key2: 234,
    // the value can be an object like a function 
    key3: function() { console.log("hello")},
    key4: {subkey1: 2, subkey2: "morning"}
};
// you can read/write the property value using the key.
obj.key2 = obj.key2 + 10; 
// you can also use [] to read/add/change a property
obj["key2"] = obj["key1"] + " is like this"; 
obj.key3();

The parameters of JavaScript functions default to undefined, but in some scenarios, it might be useful to set a different default value. Default parameters allow formal parameters to be initialized with default values. In the example below, the default value of paramter b is 1.

Every time you define a new variable using let or const, that variable can be used only within the current scope, which can be the current function block or for-loop block. For now you can think of a scope block as anything in between a matching pair of curly brackets ( {} ). Please follow the example below to understand when a variable can or cannot be used. 

  // define a in the global scope so it can be used anywhere 
  let a = 10; 
  
  function f1() {
    // define a local variable b that is only usable in function f1
    let b = 20;
    if (b > 10) {
      // define a local variable within the if block
      let c = 15;
      // b can be used since the if block is f1's child scope
      b = c + 5;
      // a can be used since a is globally visible
      a = b + 5;
    }
  
    // c can't be used here since it is only visible within the if block
    c = 20; // <-- will cause an error
  }
  
  // b can't be used here since it is only defined within the function f1's block
  b = 30; // <-- will cause an error

For more advanced information on variable and scope, you can read this well-written tutorial online.

A conditional statement is used to run certain code statement only when a particular condition is satisfied. 

  if (condition) {
    // code to run when condition is true
  } else {
    // code to run when condition is false
  }

Here "condition" must has a boolean value, either "true" or "false". It can be a comparison statement using one of the following comparison operations: equal to ( == ), not equal to ( != ), greater than ( > ), less than ( < ), greater than or equal to ( >= ), less than or equal to ( <= ).

If you know Scratch, this is equivalent to using the control block together with boolean blocks like .

When making more complicated decisions, you can combine multiple conditions with one or more of the following logical operators: "&&" (and), "||" (or), and "!" (not).
