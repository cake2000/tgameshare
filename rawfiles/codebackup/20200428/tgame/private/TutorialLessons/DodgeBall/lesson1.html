The update function

this.update is the most important function of your robot. Here this refers to the robot itself, so others can use it by calling robot.update(), similar to car.run(), or bird.fly().

As shown in the picture, the game engine collects information from the game world and then calls your robot's update function, sort of like asking "hey, here is updated information on what's going on. What do you want to do now?".

In response, your robot can send some commands back in its update function, and the game engine will execute those commands for you. This loop from step 1 to step 4 occurs very frequently (about 60 times per second) so your robot can make real-time decisions based on latest information.

Now let's start by adding an empty update function to your robot code. Please copy or type the code below into the robot code editor on the right. When you are done, say "next" or "ok" to continue.

this.update = function(info) { };
Challenge: driving the car forward

Your first challenge is to simply drive the car forward.

To do so, the robot needs to call the function sendCommand with parameter "F" in its update function.

When you are done, click the TEST button on the top right, and I expect your car to run forward and pick up the ball.

You will see a message box for test result. If you want to dismiss the game window before the test result, you can click the "Exit" button or press the ESC key at any time.

Challenge: shooting at the blue car

Now your car can pick up the red ball, the next challenge is to shoot it out, as the blue car is standing far in the front.

To shoot the ball, your robot can call the function sendCommand with parameter "S". Once you have added the command, click the TEST button to check if your robot is acting as expected.

Again, you can press the ESC key to dismiss the game window at any time.
